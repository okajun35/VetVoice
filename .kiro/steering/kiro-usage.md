# Kiro使用ガイド

## Shell コマンド実行の有効化

Kiroでshellコマンド（git、npm、bashコマンド等）を実行するには、shellツールを信頼済みにする必要があります。

### 設定方法

チャットで以下のコマンドを実行してください：

```
/tools trust shell
```

このコマンドを実行すると、Kiroがsubagent経由でshellコマンドを実行できるようになります。

### 信頼済みにすると使えるようになる操作

- Git操作（ブランチ作成、コミット、プッシュ、PR作成）
- npm/yarn コマンド（インストール、テスト実行、ビルド）
- ファイルシステム操作（mkdir、cp、mv等）
- その他のbashコマンド全般

### セキュリティ

shellツールを信頼済みにすると、Kiroは任意のshellコマンドを実行できるようになります。信頼できるプロジェクトでのみ有効化してください。

## Git専用サブエージェント

Git操作を効率化するため、`git-workflow-agent` カスタムエージェントが利用可能です。

### 使用方法

```
@git-workflow-agent ブランチを切ってコミットしてプッシュ
```

### 主な機能

- ブランチ作成・切り替え
- 適切なコミットメッセージの自動生成
- プッシュ
- PR作成（gh CLI使用）
- ステータス・差分確認

## その他のTips

### ファイル参照

チャットで `#File` または `#Folder` を使用すると、特定のファイルやフォルダをコンテキストに含めることができます。

### 問題の確認

`#Problems` を使用すると、現在のファイルのエラーや警告を確認できます。

### ターミナル出力の確認

`#Terminal` を使用すると、ターミナルの出力をコンテキストに含めることができます。

## 言語設定

Kiroとのやり取りにおける言語設定:

- **思考プロセス**: 英語で実施（効率的な処理のため）
- **回答と説明**: 日本語で提供（ユーザーとのコミュニケーション）
- **コード内**: 英語（変数名、関数名、コメント）
- **ドキュメント**: 日本語（スペック、ステアリングファイル）

この設定により、Kiroは内部処理を効率的に行いながら、ユーザーには分かりやすい日本語で説明を提供します。
